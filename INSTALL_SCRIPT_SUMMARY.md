## ✨ 主要功能

### 1. 多种安装模式

| 模式 | 参数 | 说明 | 适用场景 |
|------|------|------|----------|
| **交互模式** | 无参数 | 逐步引导用户选择 | 新手用户 |
| **系统级安装** | `-s` / `--system` | 安装到 `/usr/local/bin` | 全局使用 |
| **用户级安装** | `-u` / `--user` | 安装到 `~/.local/bin` | 个人使用 |
| **仅编译** | `-b` / `--build` | 只编译不安装 | 开发调试 |
| **仅清理** | `-c` / `--clean` | 清理编译产物 | 维护清理 |

### 2. 自动化流程

```
检查依赖 → 清理旧文件 → 编译项目 → 安装 → 验证 → 完成
```

每个步骤都有清晰的彩色输出提示。

### 3. 智能特性

- ✅ **依赖检测** - 自动检查 gcc、make 是否安装
- ✅ **错误处理** - 遇到问题时提供明确的解决方案
- ✅ **PATH 检测** - 用户级安装时检查 PATH 配置
- ✅ **桌面集成** - 可选创建 .desktop 入口文件
- ✅ **安装验证** - 自动验证安装结果并显示版本信息

## 🎨 用户体验

### 彩色输出

```bash
[INFO]    - 蓝色   # 普通信息
[SUCCESS] - 绿色   # 成功提示
[WARNING] - 黄色   # 警告信息
[ERROR]   - 红色   # 错误提示
```

### 交互式菜单

```
选择安装方式:
  1) 系统级安装 (/usr/local/bin，需要 sudo 权限)
  2) 用户级安装 (~/.local/bin，无需 sudo)
  3) 仅编译不安装
  4) 取消安装
```

## 🌟 优势对比

### vs Makefile install 目标

| 特性 | install.sh | make install |
|------|-----------|--------------|
| 依赖检查 | ✅ 自动 | ❌ 无 |
| 清理旧文件 | ✅ 自动 | ❌ 手动 |
| 用户交互 | ✅ 友好 | ❌ 无 |
| 安装模式 | ✅ 多种 | ❌ 单一 |
| 错误提示 | ✅ 详细 | ❌ 简单 |
| PATH 检测 | ✅ 智能 | ❌ 无 |
| 桌面集成 | ✅ 可选 | ❌ 无 |

### vs 手动编译安装

| 步骤 | install.sh | 手动操作 |
|------|-----------|----------|
| 检查依赖 | 1 条命令 | 多条命令 |
| 清理 | 自动 | `make clean` |
| 编译 | 自动 | `make` |
| 安装 | 自动 | `sudo make install` |
| 验证 | 自动 | 手动测试 |

## 📊 安装流程对比

### 传统方式（5+ 步）

```bash
# 1. 检查依赖
gcc --version
make --version

# 2. 清理（可选）
make clean

# 3. 编译
make

# 4. 安装
sudo make install

# 5. 验证
debpkg --version
```

### 使用 install.sh（1 步）

```bash
./install.sh --system
```
